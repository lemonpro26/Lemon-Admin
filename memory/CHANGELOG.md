# Lemon Pros — Changelog

## 2026-07-10 (fork continuation)
- **Retained tab self-heals junk/duplicate names:** The Retained endpoint only re-looked-up QB names when a record had NO name, so a record already stuck on a junk value ("SANTA ROSA CA (Duplicate)") never refreshed. Now it re-looks-up any item whose name is missing OR contains "(Duplicate)". Combined with the deterministic `_best_match` lookup, stuck names auto-correct on Retained tab load. Verified end-to-end in preview: a call stuck on "SANTA ROSA CA (Duplicate)" auto-corrected to "Jessica Martinez Escarcega".
- **Quickbase name flip-flop fixed (duplicate records):** When several QB records share a phone number, `lookup_by_phone` used `top:1` with no ordering, so a retained client's name randomly flipped between the real record and a junk "(Duplicate)" record (e.g. (707) 852-9185 → "Jessica Martinez Escarcega" vs "SANTA ROSA CA (Duplicate)"). Now fetches all matches (top:25, deduped by record id) and picks the best via `_best_match` scoring: skips "(Duplicate)" names, prefers records with an email and a proper mixed-case name. Deterministic — verified 3/3 returns "Jessica Martinez Escarcega". Existing records self-heal on next sync.
- **Funnel Analytics now counts ALL landing pages:** `/admin/funnel` and `/admin/funnel/campaigns` previously recognized only home/lapa/sp/laspa and dumped everything else (Demand Gen /dg, /dgs and Team /tm, /tm2) into "home", and didn't canonicalize aliases (/dg stores `ladg`). Fixed to canonicalize via `_canon_page` + reverse-alias map `_CANON_TO_ALIASES` and expose all 8 pages (Home, PA, PA Spanish, Spanish, Demand Gen, Demand Gen Spanish, Team Overlay, Team Split). Added matching tabs in `AdminFunnel.jsx`. Verified via curl + screenshot.
- **Quickbase lookup now matches secondary/alt phone numbers:** `lookup_by_phone` only searched the primary "Phone Number" field (fid 7), so retained clients whose call number is stored under "Alt Phone Number" (fid 47) never got their name auto-filled. Added auto-discovery of ALL phone-type fields via the QB fields API (cached) and search across them with OR. Verified live: (707) 852-9185 now resolves to "Jessica Martinez Escarcega". Existing records self-heal on the next hourly sync / "Sync now" / Retained tab reload.
- **Landing pages grouped by shared phone number (expandable):** Pages that share a tracked number are now combined into one row and their stats aggregated. Groups: "Home, PA & Team" (844-335-8911 → /, /pa, /tm, /tm2), "Spanish (/sp + /spa)" (866-524-3722), "Demand Gen (/dg)" (833-240-9312), "Demand Gen Spanish (/dgs)" (833-868-1802), plus "Direct / Untracked". Each combined row is expandable (chevron) to reveal per-page visits/leads/calls/retained/financials. Backend `_page_group`/`_GROUP_META` + `_group_landing` in `/admin/analytics`; frontend `DrillTable` gained `expandable` with nested child rows. Verified via seeded multi-child data + screenshot; test data cleaned up.
- **Duplicate landing-page rows FIXED (data bug):** The same page was stored under alias `source_page` codes (`dg`/`ladg`, `tm`/`latm`, `pa`/`lapa`, `spa`/`laspa`) which aggregated as separate rows. Added `_canon_page`/`_PAGE_ALIASES` and canonicalized clicks/leads/calls/revenue/retained/spend in `/admin/analytics` so each page yields exactly ONE row. Fixed `GROUP_PRIMARY_PAGE` to use canonical codes (`dg`/`dgs`). Verified: testing_agent iteration_24 13/13 pass, no duplicate/alias codes remain.
- **calls > visits on /spa — explained (not a bug):** The Spanish tracked number (866-524-3722) is SHARED by both /sp and /spa. Calls with no page visit/tap default to the group's primary page (/spa), so /spa absorbs Spanish-number calls beyond its own visits. Known attribution heuristic.
- **By Campaign retained reconciliation:** Retained was counted by `created_at` (leads/calls created in range) but the Retained tab counts by `retained_at`. Fixed `/admin/analytics` to count retained by `retained_at`, attribute to campaign/landing page (calls enriched for click+tap attribution), and roll retained with no campaign (or paused campaigns not shown) into a new **"Unattributed / Direct"** row so the Retained column sums to the Retained tab total. Verified (both = 2 in preview).
- **Totals row in Analytics tables:** Added a `showTotals` footer to `DrillTable` (`AdminAnalytics.jsx`). By Landing Page and By Campaign now show a bottom **Total** row. Additive columns (Visits/Leads/Calls/Spend/Revenue/Retained) are summed; ratio columns (Conv. Rate, ROAS, CPL, CPA, Bounce Rate) are recomputed from the aggregates (weighted), not naïvely summed. Verified via screenshot.
