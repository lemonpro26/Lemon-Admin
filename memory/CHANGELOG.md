# Changelog

## 2026-07-07 (cont.)
- BUGFIX (Quickbase names getting wiped): Retained clients were showing the caller's city (e.g. "LA PUENTE CA") instead of their real name even though Quickbase has the name. Root cause: `_enrich_from_quickbase` ALWAYS wrote `qb_name`/`qb_email` from the lookup result — so any failed/rate-limited/timed-out lookup during the hourly full-sync overwrote a previously-found name with a blank, and it never self-healed. Fixes: (1) only write qb_name/qb_email when Quickbase actually returns a name — never clobber a good name with an empty result; (2) Retained tab now re-attempts the lookup for ANY retained client still missing a name on every load (was: only records with no prior lookup), so nameless clients self-heal; (3) `_run_quickbase_sync` + retained enrichment use `gather(return_exceptions=True)` so one failing record can't abort the whole sweep. Verified: a wiped "C TREVISIN" record self-healed to "Chris Trevisin…" on retained load; a forced None lookup preserved the existing name. Verified lookups return names for all 3 reported numbers (626/951/858).
